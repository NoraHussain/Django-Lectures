# Django MVT architecture

(Model–View–Template)

## ما هو نمط MVT؟

ـ MVT هو اختصار لـ Model-View-Template، وهو النمط المعماري الذي يستخدمه إطار عمل Django لتنظيم وبناء تطبيقات الويب. يقسم هذا النمط التطبيق إلى ثلاثة مكونات رئيسية منفصلة ومترابطة.

## المكونات الثلاثة

### 1. ـ Model (النموذج)

- **الوظيفة:** يمثل طبقة البيانات ويتعامل مع قاعدة البيانات
- **المسؤولية:** تعريف بنية البيانات والعلاقات بينها وإجراء العمليات على قاعدة البيانات
- **مثال:** إنشاء model لمستخدم يحتوي على الاسم والبريد الإلكتروني وكلمة المرور

### 2. ـ View (العرض)

- **الوظيفة:** يحتوي على منطق العمل (Business Logic) للتطبيق
- **المسؤولية:** معالجة طلبات المستخدم، التفاعل مع models، وإرجاع الresponse المناسب
- **مثال:** method أوclass يستقبل HTTP request، يجلب البيانات من قاعدة البيانات، ويرسلها إلى الtemplate

### 3. ـ Template (القالب)

- **الوظيفة:** يمثل طبقة العرض (Presentation Layer)
- **المسؤولية:** تحديد كيفية عرض البيانات للمستخدم باستخدام HTML و Django Template Language
- **مثال:** ملف HTML يحتوي على متغيرات Django التي يتم ملؤها بالبيانات من View

## كيف تعمل المكونات معاً

1. **المستخدم يرسل طلب( request ):** يتم توجيه الطلب إلى View المناسب
2. **الـ View يعالج الطلب:** يتفاعل مع Model لجلب أو تعديل البيانات
3. **الـ Model يتعامل مع قاعدة البيانات:** يقوم بالعمليات المطلوبة ويعيد النتائج
4. **الـ View يختار Template:** يرسل البيانات إلى القالب المناسب
5. **الـ Template يعرض البيانات:** يتم عرض الصفحة النهائية للمستخدم

## مميزات نمط MVT

- **فصل الاهتمامات(Separation of Concerns):** كل مكون له مسؤولية محددة
- **سهولة الصيانة(Maintainability):** يمكن تعديل أي جزء دون التأثير على الأجزاء الأخرى
- **إعادة الاستخدام(Reusability):** يمكن استخدام نفس Model أو Template في أماكن متعددة
- **العمل الجماعي(Team Collaboration):** يمكن للمطورين العمل على مكونات مختلفة في نفس الوقت
