# Virtual environments

## معنى venv

**ـ venv** هو اختصار لـ **Virtual Environment** (البيئة الافتراضية) في لغة البرمجة Python.

### ما هي البيئة الافتراضية؟

البيئة الافتراضية هي مساحة معزولة تسمح لك بإنشاء بيئة عمل مستقلة لكل مشروع Python. هذا يعني أنه يمكنك تثبيت المكتبات والحزم الخاصة بمشروع معين دون التأثير على المشاريع الأخرى أو على نظام Python العام في جهازك.

### فوائد استخدام venv:

- **عزل المشاريع:** كل مشروع له مكتباته الخاصة وإصداراته المستقلة
- **تجنب التعارضات:** منع حدوث تعارضات بين إصدارات المكتبات المختلفة
- **سهولة النقل:** يمكن نقل المشروع بسهولة إلى أجهزة أخرى مع الحفاظ على نفس البيئة
- **الأمان:** حماية نظام Python العام من التغييرات غير المرغوبة

### كيفية إنشاء بيئة افتراضية:

```bash
python -m venv myenv
```

### تفعيل البيئة الافتراضية:

**على Windows:**

```bash
myenv\Scripts\activate
```

**على macOS/Linux:**

```bash
source myenv/bin/activate
```

### تثبيت Django في البيئة الافتراضية:

بعد تفعيل البيئة الافتراضية، يمكنك تثبيت Django باستخدام pip:

```bash
pip install django
```

### التأكد من تثبيت Django:

للتحقق من أن Django تم تثبيته بنجاح، يمكنك استخدام أحد الأوامر التالية:

**الطريقة الأولى - التحقق من الإصدار:**

```bash
django-admin --version
```

**الطريقة الثانية - عرض معلومات الحزمة:**

```bash
pip show django
```

**الطريقة الثالثة - عرض جميع الحزم المثبتة:**

```bash
pip list
```

إذا ظهر إصدار Django، فهذا يعني أن التثبيت تم بنجاح!
